# Contributing to PanoPro

## Workflow
- Do not commit directly to `main`
- Create a feature branch for any work
- Open a pull request for all changes
- Get changes reviewed before merging

## Branch naming
Use clear branch names such as:
- `feature/...`
- `fix/...`
- `chore/...`

## Pull requests
- Keep pull requests focused and small when possible
- Clearly describe what changed
- Link the related task or issue if one exists

## AI-assisted work
- AI-generated code or docs must be reviewed by a human before merging
- Do not accept large AI-generated changes without checking them carefully

## General guidelines
- Keep files organized and names clear
- Avoid unnecessary complexity
- Prefer small, understandable changes
